# moira-schedule 要件 早わかり

> requirements.md(EARS 形式の要件本文)を読む前に全体像を掴むための要約。詳細は [requirements.md](./requirements.md) へ。

## 目的

`moira-schedule` は、Moira 正典モデル `moira/MODEL.md`(SSOT) を本番アーキテクチャへ落とす **CQRS 分解の Wave1（依存: `moira-core`）** であり、Moira の **時間軸（schedule）導出** を所有する read/導出 spec である。`moira-core` が所有する書き込み契約（emit）・読み出し契約（derive）・effective-set・凍結スロット記録・二層データを **消費** し、生きた予測スケジュール・未割当バックログ・キュー・スケジュール・カバレッジ・スロット陳腐化検出・過負荷/期日超過の検出データ・D_pred とスケジュール・バッファを、同一ログ・同一構成入力から導出する。

指標式本体（EV_abs/EV%/PV/AC/SPI/CPI/sunk）は `moira-evm` が、SPI への R-S6 de-rate **適用** は `moira-evm`/`moira-health` が、9 warning の **確定/集約** は `moira-health` が所有する。本 spec は検出データと母数の提供に留める。MODEL の文言を変えず・新概念を足さない。

## 主要な要件の要点

- **（要件1）** 生きた予測スケジュールを、人間が与えた割当と依存 DAG の上での c(i,d) 平準化・クリティカルパス優先の発見的解として導出する。イベント毎に増分再計算し、未指定日の容量は既定 1.0MD/日、c=0 の日には計画作業を載せない。葉は「合意済み・割当済み・見積済み」のときにのみスケジュール可能。大域最適は追わず発見的・実行可能・非最適な解を保つ。予測導出中に凍結ベースライン・スロットには触れない。
- **（要件2）** c(i,d) 平準化を人間資源にのみ適用し、エージェント資源は容量制約の対象外とする。エージェント作業は人間接点間の実行可能スパンとして表示し、そのリードタイム（P6）を依存連鎖のパス長計算に算入してクリティカルパスへ寄与させる。
- **（要件3）** 合意済みだが被割当者を持たない有効葉を未割当バックログとして導出し、スケジュールに載せない。スケジュールは割当済み（コミット）領域のみを語り、未割当を暗黙に仮定せず可視ギャップとして公開する。
- **（要件4）** エージェント作業キューと人間レビューキューを、同一 DAG×ログへの actor フィルタ違いとして導出する（被覆上 1 導出）。エージェント作業キューは `ready`/`implementing` のエージェント割当有効葉、人間レビューキューは `implemented` で `accepted` レビュー待ちの有効葉。別々の再計算はしない。UI-ARCHITECTURE の「作業/レビュー/エージェント」三キュー表記は、この同一クエリへの actor フィルタ・プリセットであって第 3 の独立した再計算ではない。
- **（要件5）** スケジュール・カバレッジを、合意済み有効葉のうちスケジュール済み（凍結ベースライン・スロットを持つ）ものの割合として母数導出する。分母が空なら 0。SPI の計算や de-rate の適用は自身では行わない（`moira-evm`/`moira-health` の責務）。
- **（要件6）** 生きた予測完了が凍結ベースライン・スロットと乖離するサブ単位を検出し、陳腐化スロット数を原因別（割当変更・容量 c(i,d) 変更等）に提示する。乖離閾値は実装定義。陳腐化標識の解除は、理由付き再ベースラインまたは予測の再収束時のみ（意図的な「確認」では解除しない）。検出データとして提示し、警告としての確定・集約は行わない（`moira-health` の責務）。
- **（要件7）** 人間が与えた割当が c(i,d) 平準化に収まらない人物・期間（c=0 の日の AC 計上を含む）を過負荷検出データとして提示する。発火閾値・窓は実装定義。割当変更や c 変更で条件を偽化させ、点事象は窓から外れて消える。自動での再調整や要員追加は行わない。
- **（要件8）** 導出スケジュールが外部期日を超える場合、超過量（導出完了 - 期日）を伴う期日超過検出データを提示する。エージェント区間のリードタイムは要件2により既に算入済みのため超過は正しく発火する。自動での要員追加やスコープ削除は行わない。導出スケジュールが期日内に収まれば条件が偽化する。
- **（要件9）** 各サブ単位の被割当者を、作業開始ライフサイクル遷移の単一属性（latest-wins 置換）として読み取る。複数名指しが存在するときは `(ts,id)` latest-wins を現行とし、名指しを追加でなく置換として扱う。事前設定の暫定被割当者も予測上は同一に扱う。
- **（要件10）** プロジェクト導出完了 D_pred をクリティカルパス末端の予測完了（複数末端なら max）として導出する。期日（硬い上限）と目標日（計画完了の参照点）を第二層の構成入力として消費し、バッファ残量 = max(0, 期日 - D_pred)、バッファ消費率 = clamp01((D_pred - 目標日)/(期日 - 目標日)) を導出する。スケジュール・カバレッジが低い間はバッファを de-rate して対読み提示する。核心導出（EV/SPI/CPI/PV/スケジュール）は一切調整しない。バッファは D_pred とともに動く生きた予測側の量であり凍結ベースラインではない。
- **（要件11）** 期日・目標日・D_pred の有無に応じたバッファの境界条件を一意に定める。目標日なし=消費率 N/A、期日なし=バッファ未定義かつ期日超過検出を沈黙、目標日=期日=消費率 N/A（分母 0）で残量のみ、目標日>期日=構成エラー警告（自動拒否しない）でバッファ N/A、D_pred なし（カバレッジ=0）=残量を値に潰さず算出不能として可視ギャップに提示。
- **（要件12）** 凍結ベースライン（PV の源）と生きた予測スケジュール（割当・期日用に増分再計算）を別物として扱い、PMB と EAC/forecast の分離を写す。いかなる予測導出中も凍結ベースライン・スロットを再計算・上書きしない（不可侵をここに一本化所有）。凍結スロットの選定は実装依存とし、非決定性（P8）を除去でなく開示で扱う。

## スコープ

**やること（moira-schedule が所有）**
- 生きた予測スケジュール導出（c(i,d) 平準化・クリティカルパス優先・発見的増分）と各サブ単位の予測完了。
- エージェント非平準化 + リードタイムのパス長寄与。
- 未割当バックログ導出（割当なし = 可視ギャップ）。
- キュー導出（エージェント作業/人間レビュー = 同一 DAG×ログへの actor フィルタ違い、被覆上 1 導出）。
- 単一被割当者の作業開始遷移属性としての解釈（latest-wins 置換）。
- スケジュール・カバレッジの母数導出（合意済みのうちスケジュール済みの割合）。
- スロット陳腐化の検出（生きた予測 vs 凍結ベースライン・スロットの乖離、原因別カウント）。
- 過負荷の検出データ（c 平準化に収まらない人物・期間）。
- 期日超過の検出データ（導出スケジュール vs 期日、超過量）。
- D_pred（クリティカルパス末端の max）とスケジュール・バッファ残量/消費率（境界条件含む、スケジュール・カバレッジ de-rate 対読み）。

**やらないこと（上流/下流が所有）**
- emit/derive 契約・effective-set 導出・凍結スロット記録機構・ノード/見積 状態機械・二層データ永続化 = `moira-core`（上流）。
- 指標式本体（EV_abs/EV%/PV/AC/SPI/CPI/sunk）と SPI への R-S6 de-rate 適用 = `moira-evm`。
- 9 warning の確定/集約・行為列挙・clearance・at-risk(P5) = `moira-health`。
- tree+DAG/ready/orphan/restoration の表示 = `moira-scope-deps`。
- 全 write（割当 transition 発行・slot 凍結・c 改定・期日/目標日 設定・再ベースライン・リスケのオーケストレーション・人間承認）= write skill 群。
- 全 UI 提示（Gantt/DAG ビューア/付替プルダウン/CCPM フィーバー）= `moira-surface-schedule`/`moira-surface-health`。
