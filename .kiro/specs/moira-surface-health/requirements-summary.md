# moira-surface-health 要件 早わかり

> requirements.md(EARS 形式の要件本文)を読む前に全体像を掴むための要約。詳細は [requirements.md](./requirements.md) へ。

## 目的

`moira-surface-health` は、Moira 正典モデル `moira/MODEL.md`(SSOT) を本番アーキテクチャへ落とす **CQRS 分解の Wave3（read サーフェス群）**の一つで、**「健全性・EVM」軸の常駐 read-only ダッシュボード**を所有する spec である。MODEL が確定する「進捗・スケジュール・健全性はすべて 4 イベントからの導出」に対し、本 spec はその導出を**読むだけ**で、管理者が即座に健全性を把握できるよう提示する。

本 spec は UI-ARCHITECTURE §6 の read 規律に従う **read-only ダッシュボード**であり、自前の真実源・可変状態・dismiss フラグ・隠れキャッシュを持たない。指標式・effective-set 導出・平準化／D_pred／buffer 導出・warning 確定・全 write は本 spec の範囲外で、**再定義・再計算しない**（二系統計算の禁止）。本 spec が固有に担保するのは、MODEL が強制する**提示の下限**と、de-rate 規律・区別表示・対読み・可視ギャップである。

## 主要な要件の要点

- **累積EV(EV_abs) と 現行進捗(EV%) の区別ゾーン表示（要件1）**: 現行進捗（現行有効集合の EV%、R-S5）と累積EV（supersede された過去作業を含む EV_abs）を、`moira-evm` が導出する値として区別したゾーンで提示し、両者を決して混同しない。EV_abs/PV/AC は予算次元（MD）の絶対量、EV% は達成率、SPI/CPI は無次元比として次元を混同しない。累積EV は見積カバレッジと独立に提示する（基底が現行有効集合と異なるため）。見積カバレッジが低い間は EV% を de-rate し、全体完成度として提示せず可視ギャップを示す（R-S4 同型、副 host）。現行有効集合の導出は `moira-scope-deps`/`moira-evm` が所有し、本サーフェスでは実装しない。
- **PV/AC/SPI/CPI の提示と SPI の低スケジュール・カバレッジ de-rate（要件2）**: SPI・CPI・PV・AC を `moira-evm` が導出する値として表示し、本サーフェスで式を再計算しない。スケジュール・カバレッジが低い間は SPI を de-rate し、全体の対計画進捗として提示せず、未スケジュールの合意作業を可視ギャップとして示す。de-rate の**提示**は本サーフェス（R-S6 主 host）の責務であり、素の SPI 値とスケジュール・カバレッジ値は `moira-evm`/`moira-schedule` が供給する。SPI は必ずスケジュール・カバレッジと対で示す（単独表示禁止）。CPI は領域非対称を正直に提示する（分子=完了のみ、分母=仕掛含む）。SV/CV を提示する場合は正典外の提示恒等式として明示し、SV は SPI/R-S6 同型で de-rate、CV は CPI 同型で仕掛側悲観を正直に提示する。集約 SPI と R-S6 de-rate は主 host として提示し、副 host `moira-surface-schedule` と同一の導出を読む（乖離する第二の値を導出しない）。集約スケジュール・カバレッジは本サーフェスで host する。
- **SPI/CPI=N/A を潰さない（要件3）**: PV=0 のとき SPI を、AC=0 のとき CPI を、1.0 や 0 に潰さず「算出不能」として提示し可視ギャップとして公開する。各健全性メトリクスはコミット領域についてのみ語り、未コミット領域を暗黙に仮定せず可視ギャップとして公開する。
- **実行カバレッジ(R-S8) の EVM 文脈再掲と三者対読み（要件4）**: 実行カバレッジ（合意済み有効葉のうち `implementing` のノード数比率、分母0→0）を `moira-evm` が導出する値として EVM 文脈で EV% と並べて再掲する（再計算しない）。実行カバレッジを EV% と算術和して全体進捗としてはならない（仕掛中の量として提示）。三者対読み（EV%・見積カバレッジ・実行カバレッジ）の副 host として提示し、主 host は `moira-surface-spec-value`。忠実度が `implementing` の記録に依る旨を示しつつ、ライフサイクルや EV の式は変えない。
- **人別(actor 別) CPI 時系列（要件5）**: CPI を人別（actor 別）の時系列として提示し、P3（AC の actor 帰属）に立脚する上流の actor 帰属導出状態から読む（本サーフェスで再計算しない）。**人別 SPI は、分母 PV が正典上 actor 帰属を持たないため除外する**（MODEL §2.4: 計画軸=単一 assignee）。actor 別 EV_abs/CPI は `moira-evm` が新規に供給すべき導出状態であり（現 MODEL 未定義）、未供給の間は上流が提供するもののみを提示し、人別指数を自前で合成しない。valid-time の c（R-U14）が無い区間では補間・曲線捏造をせず実点のみをプロットし、過去点を再現できない場合は単一の as-of 点を正直に提示する。project-level の SPI/CPI トレンドも同一規律（実点のみ・補間禁止）。
- **スケジュール・バッファ残量/消費率(R-T6) の提示と de-rate 対読み（要件6）**: バッファ残量とバッファ消費率を `moira-schedule` が導出する値として表示し（D_pred やバッファの再計算はしない）、スケジュール・カバレッジが低い間は de-rate して対で提示する（R-S6 同型）。バッファは D_pred とともに動く生きた予測側の量として、凍結ベースライン PV/SPI とは区別する。バッファ残量と R-T4 期日超過量は同一導出の表裏（バッファ残量=超過前の監視、超過量=残量 0 クランプ後の同量）として提示し、R-T4 警告の発火・消去は行わない（`moira-health` 所有）。
- **バッファ境界条件の正直な提示（要件7）**: 目標日なし→バッファ残量のみ提示・消費率は N/A。期日なし→バッファ未定義・超過読みなし。目標日=期日→消費率 N/A（分母0）・残量のみ。目標日>期日→構成エラー警告を上流が導出したものとして提示しバッファ N/A（本サーフェスで発火・確定・消去しない。read-only）。D_pred なし（スケジュール・カバレッジ=0）→バッファ残量を算出不能として可視ギャップで提示。構成エラー警告は 9 inbox 警告には含まれず、発火・host 権限は design 送り［要検証］。
- **CCPM フィーバー的可視化と見積カバレッジ＋予算基底（要件8）**: バッファ消費率と進捗のフィーバーチャート的可視化を提示の自由（P0）として提示し、両軸は導出状態から読む（自前計算しない）。CCPM ゾーン閾値（GREEN/YELLOW/RED）は正典外の裁量パラメータ（§5）ゆえ正典として適用しない。見積カバレッジ（P2）と PV・EV を導出状態から読んで提示し、予算総額 BAC を予算基底として対で提示する。ただし BAC は MODEL §3 の R-S2 導出にも `moira-naming.md` にも正典列挙が無く、上流導出の R-S2 値とは断定しない。BAC の供給 skill と出所（凍結ベースライン予算総和か最新見積総和か）は design 送り［要検証］。バッファをノードとして表現せず（A1）、CCPM のバッファ生成（安全余裕除去・末尾集約）は採らない。
- **警告の提示と de-rate 型 vs 判断型の区別（要件9）**: R-S6 de-rate 信号は常時メトリクス修飾として本サーフェスに提示し、decision インボックスへルーティングしない。R-T4 期日超過警告と R-S3 thrashing 警告は `moira-health` が確定したものとして surface し、decision インボックス／write 文脈へ深リンクする（本サーフェスで確定・消去しない）。いかなる警告にも acknowledge/dismiss/既読の可変状態を提供しない（条件が真の間は持続し、入力が偽化した時にのみ消える。§2.1）。警告を抑制・畳む場合も可視ギャップの会計から落とさない（P0）。構成エラー警告（目標日>期日）は de-rate 型でも 9 inbox 警告でもなく、上流が導出したものとして提示のみ（発火・host 権限は design 送り［要検証］）。
- **read-only ダッシュボード規律（要件10）**: 全値を `moira-core` の単一 R-S2 導出から読み、他ダッシュボードと同一のクエリを参照する。自前の真実源・可変な導出キャッシュ・dismiss フラグを持たず、同一概念に対し乖離する第二の値を導出しない。上流で導出状態が更新されたとき、サーフェスごとの二度目の再計算を行わず反映する。write は所有せず、write 文脈への深リンクのみ提供する（R-T4 のスコープ/期日判断は health と schedule-time の両文脈を要するため複数 deep-link を提供し片面飛びにしない）。共有視覚部品は全サーフェス共通の theme モジュールから描画し、独自に再実装しない。各警告の行為列挙は `moira-health` 導出層から読み、本サーフェスで再定義しない（深リンク発信元のみ）。役割（管理者/開発者）は同一 R-S2 導出に対する actor 種別のプリセット/フィルタとして扱い、役割で物理分割して別系統の値を導出しない（「二つの真実」禁止）。per-task EVM 内訳は二段を区別する: (a) 集約 derive() 所有指標は再計算しない、(b) per-node 属性射影（frozenBudget/ac から表示用に PV/EV を合成し SV/CV を差分として正典外の提示恒等式と明示する射影）は read-only の表示用写像として許容する。

## スコープ

**やること（本 spec が所有 — 提示と読みの規律）**
- **健全性・EVM 軸の read 提示**: EV_abs（累積EV）と EV%（現行進捗）の区別ゾーン表示（R-S5）、PV/AC/SPI/CPI の絶対量提示（§3）、SPI の低スケジュール・カバレッジ de-rate（R-S6、主 host）、CPI の領域非対称の正直な提示、SV/CV の提示恒等式扱い。
- **実行カバレッジ（R-S8）の EVM 文脈再掲**: 三者対読み（EV%・見積カバレッジ・実行カバレッジ）の副 host。算術和禁止。
- **人別（actor 別）CPI 時系列**: P3（AC の actor 帰属）に立脚、valid-time c 制約下で実点のみ。人別 SPI は非対象（PV の actor 帰属が正典に無い）。
- **スケジュール・バッファ残量/消費率（R-T6）**: スケジュール・カバレッジ対 de-rate、境界条件（目標日なし/期日なし/目標日=期日/目標日>期日/D_pred なし）、R-T4 表裏。
- **CCPM フィーバー的可視化**: 消費%×進捗%。ゾーン閾値は正典外。
- **見積カバレッジ＋予算基底（PV/EV/BAC）**: BAC の供給・出所は design 送り。
- **提示規律**: SPI/CPI=N/A を潰さない、可視ギャップ（R-U9/P0）、de-rate 型 vs 判断型の区別、read-only 規律（自前状態なし・二系統計算禁止）、共有視覚部品の再利用。

**やらないこと（下流/上流/基盤が所有）**
- 指標式本体（EV_abs/EV%/累積EV/見積カバレッジ/実行カバレッジ/PV/AC/SPI/CPI/sunk）= `moira-evm`。
- 平準化(P7/P8)・予測・D_pred・スケジュール・カバレッジ・buffer 残量/消費率の**導出** = `moira-schedule`。
- tree+DAG/effective-set/ready/orphan/restoration の導出 = `moira-scope-deps`。
- 9 warning の確定・集約・行為列挙の単一定義 = `moira-health`。判断型警告の集約・ルーティング = `moira-surface-decision`。
- ノード木/トレーサビリティ/見積カバレッジ行明細の主 host・三者対読みの主 host = `moira-surface-spec-value`。
- Gantt/担当/未割当バックログ・SPI/R-S6 de-rate の副 host = `moira-surface-schedule`。
- 全 write（スコープ/期日判断・再ベースライン・c 宣言等の人間コミット判断）= moira-* write skill 群。
- emit/derive・二層データ・effective-set 導出・latest-wins・状態機械・凍結記録・R-S2 オーケストレータ = `moira-core`。
- CCPM のバッファ生成（安全余裕除去・末尾集約）・ゾーン閾値 = 正典外（MODEL §5）。
