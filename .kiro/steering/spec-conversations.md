## 会話ログ

このプロジェクトでは、cc-sdd で spec を作る過程の Claude との議論(論点・選択肢・決定・理由)を、
要件や設計判断などの**項目単位**で「会話」として保存し、spec-viewer から辿れるようにする。
**この見出し(`## 会話ログ`)を含む `.kiro/steering/spec-conversations.md` の存在そのものが、
会話保存機能の有効化マーカーである。** これを貼れば、以後のフロー(通常 Claude)が本規約に従う
(人手は規約の初回設定のみ)。

### 有効化で効くこと
- 保存提案: 会話が長くなったとき、Claude が「現在の会話を会話ログとして保存しますか」と提案する
  (実行は常に人間の承認が起点。自動保存はしない)。
- 紐付け: 保存時、紐付け対象(spec / 要件 / 設計判断 / 横断・多対多)を推定提案し、人間の承認・修正に従う。
- 文脈復元: 別セッションで同じ spec・要件に着手するとき、保存済みの会話を参照してから進める。

### 配置・命名
- 配置: `.kiro/specs/{feature}/conversations/`(spec/要件/設計判断)・横断は `.kiro/conversations/`。
- 命名: `{YYYY-MM-DD}-{slug}.md`(slug は半角の `[A-Za-z0-9._-]`)。

### 保存内容(既定でこの厚みにする — 充実は Claude の責務。人手は承認のみ)
front matter(`created_at` / `title` / `bindings`[{ kind, id }] / `source`)＋本文。
「短く済ませる」より「後から判断を再構成できる」を優先する(要点に圧縮しすぎない)。
- `## 要約`: 論点 / 選択肢 / 決定 / 理由。さらに **`### 覆った判断`** に覆した・訂正した点を残す(履歴性)。
- `## 生ログ(該当範囲)`: 実際のやり取りを `- **あなた**:` / `- **Claude**:` のターン形式で、
  推論を再構成できる厚み(数ターン以上)で残す。

#### スケルトン(これを埋めて 1 会話 = 1 ファイルで保存)
    ---
    created_at: 2026-06-14T03:00:00Z
    title: (何を決めた会話かが一行で分かるタイトル)
    bindings:
      - { kind: requirement, id: "2" }
      - { kind: design-decision, id: "(設計判断キー。不要なら削除)" }
    source: claude-code-session
    ---

    ## 要約
    - 論点: (何が争点だったか)
    - 選択肢:
      - A. (案A)
      - B. (案B)
    - 決定: (何に決めたか)
    - 理由: (なぜそう決めたか)

    ### 覆った判断(履歴があれば。無ければ節ごと削除)
    - (一度こう決めた → 指摘・事実により こう覆した/訂正した)

    ## 生ログ(該当範囲)
    - **あなた**: (発言)
    - **Claude**: (提示した選択肢・根拠)
    - **あなた**: (次の発言・指摘)
    - **Claude**: (対応・結論)

同一対象への新たな会話は上書きせず別ファイルで時系列に追記する(覆りも履歴として残す)。
