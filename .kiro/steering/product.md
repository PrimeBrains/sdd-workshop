# プロダクト: Moira（モイラ）

Moira は、**仕様駆動 × チケット駆動 × EVM（Earned Value Management）を統合する管理基盤**である。プロジェクトを「見積を持つノードの木と、ポリシー付き辺の DAG の上を流れる追記専用イベント列」として捉え、進捗・スケジュール・健全性をすべてそこからの**導出**として提示する。システムは観測・導出・警告に徹し、コミットメントを伴う判断は人間に残す。

一文定義の正典は `moira/MODEL.md §0`。名称の由来 μοῖρα=「割り当てられた持ち分」= Earned Value／予算配賦 は `moira/NAMING.md`、語彙は [moira-naming.md](moira-naming.md)、モデル制約は [moira-model.md](moira-model.md) を見る。

## 中核思想: システムは警告し、人間が決める

Moira の価値の中心は **「システムは警告し、人間が決める」**（`moira/MODEL.md §2.1`）。

- 導出メトリクスは**コミット済みの領域のみを語り**、未コミット領域は常に**可視のギャップ**として示す（隠さない；P0）。
- 警告は**現在の導出状態に対する述語**であり、条件が真である限り残る——「見て受容するだけ」では消えず、**条件を偽化する入力（4 イベントの追記 または c の変更）でのみ消える**（詳細は [moira-model.md](moira-model.md)）。

## 五つの人間専任コミット判断

以下の 5 つは**システムが自動化せず、必ず人間が下す**（`moira/MODEL.md §2.1` の #1–5。この事実の steering 内 restate は本ファイルが所有し、他ファイルはここを指す）:

1. **見積の合意** — AI が提案し、人間が `agreed` へ確定する（A5）。
2. **割当** — 誰がどのタスクをやるかを人間が与える（§2.4）。
3. **スコープ／期日の決定** — 期日超過時に要員追加か機能削減かを人間が決める（R-T4）。
4. **計画の精度（見積の深さ）** — どこまで見積もるか・ノード化するか畳むかを人間が決める（§2.3, R-E2b）。
5. **人間ごとの容量 c の宣言（契約割当）** — c の契約割当（reason=契約）の改定を人間が与える（A4）。暦由来の改定（祝日・休暇）は可用性の入力であり、コミット判断は契約割当の決定を指す。

## 対象ユーザーと使い所

人間内部の権威・オーナーシップはモデルのスコープ外であり（§2.1）、閲覧者の役割は権限の分割ではなく**同じログ・同じ導出を異なる観点で読むフィルタ**にすぎない（actor〔人間／エージェント〕単位のキュー・フィルタは P4）。役割プリセット（開発者・管理者等）の整理は派生設計文書 `moira/UI-ARCHITECTURE.md` にある。具体的な利用シーンは検証シナリオ S1–S13 がカタログ化しており、複製せずそれを参照する（正典: `moira/validation-scenarios.md`）。検証シナリオのうち最初に実装されたのは S4「健全性のリアルタイム把握」（参照実装の所在は [tech.md](tech.md)／[structure.md](structure.md)）。

## スコープ外（意図的に持たないもの）

最小性のため、Moira は次を**持たない**（`moira/MODEL.md §5`「意図的に持たないもの」）:

- 警告の acknowledge／dismiss／既読（可変状態）——**根拠は最小性**（A2 ではない）。条件が真の警告は可視ギャップとして持続し、顕著さの制御は提示の責務（§2.1）。
- 重み・留保率・能力（スキル・習熟度）のモデル化——全廃（c は外的事実入力であり例外）。
- 人間内部の権威・オーナーシップ——スコープ外。矛盾合意は警告のみ（R-U12）。
- 金銭通貨——コストは人間アテンション時間（MD）の単一通貨（A6）。
- ペアリング／モブの構造表現、日内（時刻帯）スケジューリング、未発見作業の測定（原理的に不能；P2）。

## 既存プロトタイプとの関係

旧 `evm-studio` / `sdd-dashboard` は**使い捨てプロトタイプ実験**であり、Moira はその失敗を踏まえた完全新規・別概念である（`moira/README.md`）。それら旧アプリ向けの spec・steering は本リポジトリから除去済みで、Moira の確定ナレッジは `moira/`、その正典は `moira/MODEL.md` にある。
