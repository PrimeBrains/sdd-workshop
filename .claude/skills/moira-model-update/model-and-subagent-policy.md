# モデル・サブエージェント方針 / Model & Subagent Policy

本スキルは**判断を担う役割（著者・敵対者・採点者）に高能力を集中し、機械的な役割（事実照合）は
安価なモデルに出す階層構成**で動く。深さ（同一ベンダーの多重反復）でなく**ベンダー多様性と早期
引き上げ**で識別力を確保する（`review-gate.md` の停止規則と対）。
ただし「強制できないもの」を「強制する」と偽らない（正直さ）。

## 前提モデル・effort（強く推奨。ただし機械強制はできない）

理想は **主コンテキスト（著者・orchestrator）が Fable 5 または Opus 4.8 以上、reasoning effort
max** で走ること。しかしこれは
**機械的に検証できない**:
- モデルは自分の正確なバージョンを誤認しうる（自己申告は信頼できない）。環境に "exact model ID"
  が注入されていれば参考にできるが、無い環境もある。
- reasoning effort の現在値は実行時に観測する手段が無い（API/env から読めない）。

したがってこれは **HARD な自動停止ゲートにはしない**（できない）。代わりに:
- **起動時に推奨条件を明示し、ユーザーに確認を促す。** 環境に "exact model ID" があり、それが
  Opus 4.8 未満だと判る場合は、その事実を告げて `/model` 切替を勧める。判らない場合は「現在の
  モデル/effort を確認のうえ、Opus 4.8 + effort max を推奨」と正直に伝える。
- 条件を満たすかの最終責任は**ユーザー**にある（best-effort 宣言）。スキルは確認を促すまで。

この格下げは、敵対的レビューが「低性能・低effort で動いている時にこそ自己判定が効かず、HARD 宣言が
空文になる」と突いた結果であり、**嘘の強制力を主張しない**ための正直化である。

## サブエージェント（ここは実際に制御できるので強制する）

- **役割別のモデル階層で起動する。** 判断を担う役割は高能力、機械的な役割は安価に:
  - **敵対者（`moira-adversary`）・採点者（`moira-gate-judge`）= `model: opus`**（最新 Opus
    エイリアス。本環境ではこれが Opus 4.8 に解決される。**4.8 をより細かく pin する手段は Agent
    ツールに無い**ため、`opus` 指定が達しうる最善である——この粒度限界を正直に明記する）。各
    エージェント定義（`.claude/agents/moira-*.md`）の frontmatter にも同じ model を持たせ、呼び出し
    側でも明示して二重に担保する。
  - **事実検証（`moira-fact-checker`）= `model: sonnet`**。語源・数式・文献の裏取りは検索と突合が
    主体で、設計判断をしない役割のため sonnet で足りる（判断が要る争点は敵対者・採点者側の仕事）。
    sonnet 化で拾えなくなった事実誤りが後工程で発覚した場合は、この割当を見直す（`review-gate.md`
    の撤回条件と同じ反証可能性の型）。
- 派遣プロンプトで毎回**「最大の徹底度で思考し、真に敵対的に振る舞え。不確実なら『破れている/
  反証された』側に倒せ」**を課す。
- **独立性**: 同一ラウンドの `moira-adversary` は**並列**に起動し、互いの結論を見せない。著者（主
  コンテキスト）・敵対者・採点者（`moira-gate-judge`）は別主体に保つ（`review-gate.md` の三者分離）。
- レビューワーには **Write/Edit を与えない**。パッチは主コンテキストのみが行う。

## 敵対者構成（ベンダー多様性を深さの代わりにする）

同一ベンダーの敵対者は盲点が相関し、2 体目・3 体目の限界収穫が急減する。1ラウンドの攻撃枠は
**既定 = Claude `moira-adversary` 1 体＋codex レビュー 1 本**（独立ベンダー・並列）とする:

- **codex の位置づけ**: 独立ベンダーの敵対者。呼び出しは codex CLI（`npx` で **latest** を使う——
  バージョンを pin しない。過去に特定版〔0.61 系〕が動作しない事例が観測されている・⚠️会話由来の
  運用知でリポ内一次資料なし）または codex プラグイン経由。**read-only**（書き換え権限を
  与えない——パッチは著者のみ、の三者分離をベンダー越しにも保つ）。codex の指摘は著者が
  Critical/Important/Suggestion/FYI の重大度語彙に写像して記録し、Claude 敵対者の指摘と同格に
  `moira-gate-judge` の判定対象へ含める。**写像の監査**: codex の生指摘全文をラウンド記録に残し、
  著者の写像表では**ベンダー自尺度からのあらゆる格下げ**（Critical→Important を含む）に書面の理由を
  付す。採点者が生指摘と写像表を突合し、**理由なき格下げはベンダー原重大度で扱う**——Critical→
  Important の理由なき格下げは Critical 生存（deferred 不可）、非ブロッキングへの理由なき格下げは
  Important 生存（著者が写像で握り潰す・deferred 可能圏へ逃がす抜け穴を両方塞ぐ）。
- **増強**: MODEL 本体の公理・制約・イベント意味論に触れる変更など最重要対象でユーザーが明示した
  場合のみ、Claude 敵対者を 2 体（攻撃角を変えて）＋codex 1 本まで増強してよい。それ以上の多重化は
  限界収穫が正当化しない。
- **codex 不在時の fallback**: codex が使えない環境では Claude 敵対者 2 体（攻撃角を変えて）で代替し、
  ベンダー多様性が欠けたことを報告に正直に明記する。

## ラウンド内コスト上限（暴走防止）

`review-gate.md` の上限ラウンドに加え、**1ラウンド内**のサブエージェント呼び出しにも上限を設ける:
- 攻撃枠: 上記の敵対者構成（既定 Claude 1＋codex 1、増強時 Claude 2＋codex 1）。
- **反証の再反論**（`review-gate.md` 基準2）: 著者の反証に **1:1** で起動、**最大 K=3 体**（1ラウンドの
  反証は最大 3 件・超過する Critical/Important は反証でなく**修正で塞ぐ**）。攻撃枠とは別枠。
- **相反裁定のディベート**: さらに別枠で最大 **2 体**まで（同一ラウンド内）。
- `moira-fact-checker`: 同一主張群に対する反復は最大 **3 巡**。3 巡で `NO_OBJECTION` に至らない
  主張は **UNVERIFIABLE 扱い**として §7・報告に残し、膠着でラウンドを溶かさない。
- `moira-gate-judge`: 1ラウンド **1 体**。

## 名指し派遣する標準エージェント

| エージェント | subagent_type / 呼び出し | model | 用途 |
|---|---|---|---|
| 敵対的レビューワー | `moira-adversary` | opus | 穴出し。対象種別に応じ `adversarial-vectors.md` の該当節（モデル攻撃角／スキル攻撃角）を読ませる。多者ディベートは発散立場で並列起動して兼ねる |
| 独立ベンダー敵対者 | codex CLI（npx latest）/ プラグイン | GPT 系 | 穴出し（Claude と相関しない盲点）。read-only。指摘は重大度語彙に写像して judge の判定対象に含める |
| 独立採点者 | `moira-gate-judge` | opus | falsifiable ゲートの PASS/FAIL 判定、反証の健全性・相反指摘の裁定・Important の deferred 可否 |
| 事実検証 | `moira-fact-checker` | sonnet | 語源・神話・EVM 理論・数式等の裏取り |

## ワークフロー併用（任意）

ユーザーが多エージェントオーケストレーション（"ultracode" / "ワークフローを使って" 等）を明示的に
求めた場合に限り、Workflow ツールでラウンドをパイプライン化してよい。既定は Agent ツールによる
逐次ラウンドで足りる。
