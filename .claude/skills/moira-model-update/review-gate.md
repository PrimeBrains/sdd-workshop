# 確定ゲート / Falsifiable Review Gate

正典モデル（または本スキル自身・エージェント定義など Moira の確定設計物）の変更を**確定して
よいか**を決める唯一の基準。6次元自己採点は**廃止**（自己採点インフレで識別力がない儀式だった
ため）。代わりに、独立した主体による falsifiable な合否で判定する。

## ゲート基準

次が**すべて**満たされたときのみ確定してよい:

1. **反論されない Critical がゼロ。Important は全件 disposition 済み。**
   - **Critical**: `moira-gate-judge` が、敵対者の全 Critical 指摘について「修正済み」または
     「明示的反証（健全）」で決着していると判定すること。1つでも生存していれば確定不可。
     **Critical に deferred はない**——ゼロ以外の決着を認めない。
   - **Important**: 各 Important が「修正済み」「明示的反証（健全）」「**追跡付き deferred**」の
     いずれかで決着していること。**ゼロまでの反復は要求しない**（下記「停止規則」の sweet-spot
     運用の核）。追跡付き deferred の要件（すべて必須）: 後続 issue への参照＋owner＋再評価条件を
     持ち、その後続 issue が open で生きていること。**openness は著者が証跡（`gh issue view` 出力等）を
     ラウンド記録に添付して示す**——採点者は証跡で検査し（採点者自身は GitHub を照会できない粒度限界を
     正直に認める）、最終確認は確定時の人間が担う。単なる「理由付き見送り」や acknowledge は
     disposition ではなく**生存扱い**。
   - **deferred 不可の歯止め**: 当該 Important が対象の**目的達成に必須の欠陥**（放置すると正典の
     主要な制約・語彙・導出が誤って機能する）なら deferred を認めない——それは実質 Critical であり、
     採点者が Critical へ再分類して扱う（「何でも deferred」への退化を塞ぐ）。
   - deferred にした Important は §7 と確定報告に**全件列挙**する（黙って落とさない）。
2. **反証は同一ラウンド内で再反論を経ている（反証で退けるのは最大 K=3 件、超過は修正で塞ぐ）。**
   著者（主コンテキスト）が Critical/Important を「反証」で退ける場合、その反証文を**同じラウンド内で
   別の `moira-adversary` に差し戻して再反論させる**。次ラウンドに先送りしない——先送りすると、上限
   ラウンドで初めて出た反証が再反論を受けられず構造的に永久生存し、確定不能になるため。再反論で
   覆らなかった反証のみ「健全」と認める。著者の作文力だけで反証が通る経路を塞ぐ。
   **1ラウンドで著者が反証で退けられる Critical/Important は最大 K=3 件**（`model-and-subagent-policy.md`
   の再反論枠と一致）。各反証は同一ラウンド内で 1:1 に別の `moira-adversary` へ差し戻し、必ず再反論を
   受けさせる。**K 件を超える分は反証でなく修正で塞ぐ**——これで「全反証が再反論を受ける（デッドロック
   なし）」と「再反論体数に上限がある（暴走なし）」を両立させる。修正で塞ぐ件数に上限はない。
3. **相反指摘は裁定で両方が決着する。**
   並列敵対者が同一箇所に相反する Critical/Important（例: 一方「冗長で削れ」、他方「不可欠で残せ」）を
   出した場合、著者が独断で採らない。`moira-gate-judge`（必要なら発散立場の `moira-adversary`
   ディベート）が、または当該選択が人間の意図に属するならユーザーが（FORK 経由で）、いずれかを採る。
   **裁定で採られなかった側は「裁定により棄却」として決着扱いとし、生存 Critical には数えない**
   （両立不能な二つの Critical を同時要求して永久 FAIL に陥らせない）。裁定は棄却理由を明示すること。
4. **genuine な分岐がすべてユーザーに回されている。**
   敵対者が `FORK` として挙げた、または採点者が分岐と認めた**人間の意図に属する判断**（事実・論理
   では決められない選択）を、主コンテキストが勝手に決めず AskUserQuestion でユーザーに渡していること。
5. **停止性が保たれている。**
   上限ラウンド（既定 3、`--rounds N` で上書き可、ただし **N ≥ 2** を強制。N<2 は拒否）に達して
   打ち切る場合、未決着の Critical または disposition されない Important が残っているなら**確定しない**
   （打ち切りを PASS にしない）。残課題を §7 とユーザー報告に正直に出し、人間へ上申する。

判定は二値: `PASS` か `FAIL`。中間はない。迷ったら FAIL。

## 停止規則（sweet spot で引き上げる）

独立攻撃の限界収穫は急減する（実測: #39 DFD レビューは第1ラウンドで新規 16 件 → 第2ラウンドで
新規 4 件〔いずれも Important・同 DFD §10 決着表基準〕と減衰し、2 ラウンドで実用品質に達した）。
本ゲートは「同一ベンダーで深く
反復する」のでなく「**多様性 × 早期引き上げ**」で識別力を確保する（敵対者構成は
`model-and-subagent-policy.md`）:

- **早期終了**: あるラウンドで**新規 Critical = 0**、かつ**新規 Important が同一ラウンド内で全件
  disposition** された場合、採点者の PASS をもってループを終了してよい。それ以上のラウンドは義務で
  ない（追加ラウンドの限界収穫 < コストと判断する——これが sweet spot の定式化）。**「新規」の判定は
  採点者が行う**——前ラウンド指摘の再出現（言い換え・部分修正後の同旨再指摘を含む）は新規ではなく
  **同一指摘の生存**として扱う（表層パッチで「古い指摘」に見せて早期終了する経路を塞ぐ）。
- **上限 3 ラウンド**（既定。基準5）。上限到達で未決着が残るなら確定せず人間へ——これは新しい妥協で
  はなく、既存の「打ち切りを PASS にしない」規則への早期到達である。

## 撤回条件（この経済学自体の反証可能性）

Important の disposition 制・敵対者のベンダー多様性構成・3 ラウンド上限は、コスト対識別力の
sweet-spot 仮説に基づく（2026-07-19 人間批准・issue #40）。この仮説は反証可能にしておく:
`.kiro/postmortem/defects.md` に「**deferred にした Important が実害化した**」ことを根本要因とする
欠陥が未レビューで 2 件積み上がったら（`kiro-postmortem-review` の同件閾値と同型）、本ゲートの
経済学を再審査し、必要なら締め直す（Important ゼロ必須への復帰・上限引き上げ）。
**運用規律（トリガの実行可能化）**: 該当欠陥を `/kiro-postmortem-add` で記録する際、「根本要因詳細」
フィールドに literal タグ **`[deferred-important]`** を含める。再審査トリガは同タグを持つ未レビュー
（`Status: recorded`）entry が 2 件に達したこと——検知は grep で可能だが、起動は記録・照合の運用規律で
あり自動ではない（正直開示。タクソノミーのラベル一致に依存しない専用トリガとしてここに固定する）。

## 収束の意味（正直化）

「新しい Critical/Important が出なくなった」は**未来に対する全称命題で原理的に検証不能**。
よって本ゲートは「もう永久に穴が出ない」を主張しない。**最終ラウンドで独立採点者が PASS を出した
こと**を確定条件とし、それは「その時点の独立した攻撃を生き延びた」という有限で falsifiable な
主張に限定する。過剰な精度（「16ラウンド級」等）を謳わない。

## 三者分離（自己採点禁止）

| 役割 | 主体 | 権限 |
|---|---|---|
| 著者 | 本スキルの主コンテキスト | 対象を書く/パッチする |
| 敵対者 | `moira-adversary`（並列・独立） | 穴を出す／反証を再反論する。書き換え不可 |
| 採点者 | `moira-gate-judge` | 合否・反証の健全性・相反裁定。書き換え不可。著者・敵対者と別主体 |
| 事実検証 | `moira-fact-checker` | 事実主張の裏取り。書き換え不可 |

同一主体が「書いて・攻撃して・合格を出す」ことを禁じる。これがゲートの識別力の源泉。

## 重大度（敵対者・採点者で共通）

- `Critical`: 設計を破る矛盾・除去可能な中核要素・誤った事実主張・必要十分性の崩れ・手続きの破綻
- `Important`: 確定前に決着（disposition）を要する欠落・曖昧さ・未処理ケース——修正が原則、追跡付き deferred は例外枠（ゲート基準1）
- `Suggestion`: 非ブロッキングの改善（ゲートをブロックしない）
- `FYI`: 参考情報（ゲートをブロックしない）

Suggestion/FYI は確定を妨げないが、§7 や報告に残してよい。

## 確定の不変条件（部分適用の禁止）

確定（コミット）は**全ステップ完遂で一つの不可分な単位**として扱う。MODEL 系なら「版ヘッダ・§6
来歴・§7 確認事項」の三者が**互いに整合した状態**でのみ確定済みとみなす。途中で停止した場合は
**未確定**として扱い、版ヘッダだけ上がって §6 が古い、といった部分適用を残さない（残ったら次回起動
時に不整合として検出し是正する）。

## §6 来歴の評価言語と、既存来歴の扱い

- 確定時に MODEL §6 へ追記する来歴行は、「6次元満点」ではなく
  **「独立採点者の残存 Critical = 0・Important 全件 disposition（deferred があれば件数と追跡先）／
  反証された主要指摘」**で記す。
- **追記専用が原則**: 既存（v1→v9）の旧来歴行は通常**遡及改変しない**。
- **初回正典化に限る例外**: 正典を初めてこの体系に取り込む（ブートストラップ）ときに限り、旧
  「6次元満点」評価語・round 数の記述誤り（例: 「16ラウンド」と "ten rounds" の不一致）など、
  **既知の誤り・撤回済み評価語を一度だけ是正してよい**（adversarial-vectors V3「自分の過去主張も
  撤回する」と整合）。是正したことは新規来歴行に明記する。以後は追記専用を厳守。
