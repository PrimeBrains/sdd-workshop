# 対話セッション

`atelier/METHODOLOGY.md` に基づく「ソクラテス式知識抽出」セッションを運営する。
ユーザーがビギナー開発者として質問し、Claude が SDD 経験者として答える対話を
`atelier/dialogues/` にログとして蓄積することが目的。

## 1. セットアップ

### 既存ログを確認する

```bash
ls atelier/dialogues/
```

既存ファイルがある場合は「継続」か「新規セッション」かをユーザーに確認する。

### トピックを選ぶ

以下の一覧をユーザーに提示して選択させる。
既存ファイルがあるものには「(継続可)」と表示する。

| ファイル名 | モジュール |
|-----------|----------|
| `why-sdd.md` | Module 1: なぜ仕様駆動開発か |
| `workflow.md` | Module 2: cc-sdd ワークフロー全体像 |
| `forward-flow.md` | Module 3: Forward flow（新規機能） |
| `steering.md` | Module 5: ステアリング強化 |
| `maintenance.md` | Module 6: 仕様変更と保守 |
| `test-review.md` | Module 7: テスト・レビュー |
| `team-ops.md` | Module 8: チーム運用 |

ユーザーが上記以外のトピックを希望した場合は、適切なファイル名を一緒に決める。

### ペルソナを確認する

デフォルトペルソナを提示し、ユーザーに確認または変更させる:

> 「Java歴3年、普段は Spring Boot で API を作っています。Claude Code は触り始めたばかりで、SDD という言葉は聞いたことがある程度です。」

ユーザーが変更を希望した場合はそれを採用する。ペルソナが固定されると回答の粒度が揃い、後でコンテンツに変換しやすくなる。

## 2. ログファイルの初期化

`atelier/dialogues/{topic}.md` を作成する。
継続セッションの場合は既存ファイルに追記（後述の「継続セッション」を参照）。

**新規の場合:**
```markdown
# 対話ログ: {トピック名}

**セッション日**: {今日の日付}
**ペルソナ**: {確認したペルソナ}

---
```

**継続の場合:**
既存ファイルの末尾に以下を追記する:
```markdown

---
*(セッション継続: {今日の日付})*
---
```

## 3. セッション開始の宣言

ファイル初期化後、以下のように宣言してセッションを開始する:

> 「セッションを開始します。**{ペルソナの一言まとめ}** として自由に質問してください。
> 「終了」と入力するとセッションを締めくくります。」

## 4. Q&A ループ

### 各ターンの処理

ユーザーの質問を受け取るたびに:

1. **SDD 経験者として回答する**（下記の「回答のスタンス」を参照）
2. 回答後すぐに `atelier/dialogues/{topic}.md` の**末尾に追記**する:

```markdown
**Q**: {ユーザーの質問（原文のまま）}

**A**: {回答内容}

---
```

**追記ルール（厳守）**: 必ずファイル末尾に追加する。先頭・途中への挿入は禁止。
ログは時系列順（古いものが上、新しいものが下）で読めることが重要。

リアルタイムで書き込む理由: セッションが途中で終わっても対話が失われない。

### 回答のスタンス

- **断言する**: 「たぶん〜」ではなく「〜です。なぜなら〜」で答える
- **誤解を歓迎する**: 「その誤解はよくあります」と自然に切り返し、なぜ誤解しやすいかも説明する
- **具体例を使う**: 抽象論だけでなく、実際のコードやフォルダー構成を示しながら説明する
- **Why を省略しない**: SDD の「なんとなくこうする」を言語化するのがこのセッションの価値

## 5. セッション終了

ユーザーが「終了」「セッション終了」「end」「おわり」などと入力したら終了処理を行う。

### Insights の抽出

セッション全体を振り返り、以下のセクションをファイル末尾に追記する:

```markdown
## Insights（抽出メモ）

**誤解パターン:**
- {セッション中に出た誤解・前提のズレ}

**理解の転換点:**
- {「そこか！」という質問・回答のペア}

**使えそうな比喩・表現:**
- {ワークショップで使えそうなアナロジーや言い回し}

**ドラフト化候補:**
- {atelier/drafts/ に切り出すと良い素材}
```

### 次のアクションを提案する

終了後、以下のいずれかを提案する:
- `atelier/drafts/{topic}.md` への抽出作業（METHODOLOGY.md の「抽出フェーズ」）
- 別トピックの対話セッション開始
