## 確定済みの分岐(原セッション v1–v9)

- **重みテーブル → 単一式に溶解。** EV はどの階層でも「完了 ÷ 合意済み」。状態に重みを割る発想を廃止。
- **spec 留保率 → ノード化で溶解。** 仕様作業は独自見積を持つただのノード。
- **能力トレース → 廃止。** スキル・習熟度をモデル化しない。割当は人間が外から与える入力(R-U6)。
- **見積 = 合意状態を持つ提案。** `proposed→agreed`、合意は人間のみ(A5/R-U4)。粒度はタスク/feature ノード単位(仕様方法論の成果物承認ゲートと一致——cc-sdd では tasks approved——ボトルネック回避;v16 Phase 0 0c で方法論非依存に一般化)。
- **EV は合意済み見積のみで算出**、未見積は見積カバレッジで可視化(P2/R-S4)。
- **辺ポリシー既定 = implemented**、かつ `implemented ≡ 実装完了 ∧ 仕様FIX`(R-D2)。
- **「見積の見積」→ 規則の底打ちでなく人間のコミット判断(P0 第4項)に還元。**
- **スコープ削除の自動化はしない**(R-T4:アラートのみ、判断は人間)。

## 確定済みの分岐(v12 → v13: 暦/容量の単一 c 一般化)

> 「暦(祝日/休暇/稼働日)と α_i の関係」の対話から派生。moira-model-update の敵対ループ
> (adversary ×3 + fact-checker → ユーザー裁定 FORK → 再challenge ×2 → gate-judge)を経て v13 に確定。

- **【ユーザー裁定 FORK】容量の表現形 → 単一 c(α は導出 view)を選択。** 敵対者が「二因子 α×暦 はどの導出も積しか読まず冗長、かつ非一様な日次配分で a>1 を要求して破綻」と示し、AskUserQuestion で「単一 c／二因子／v12 維持」を提示。ユーザーは**単一 c** を選択。契約レート α_i は「c の reason=契約 成分」の**名前付き導出 view**(SPI/EV%/CPI と同型)へ降格——v12 で公理化した α_i の一級位置づけを撤回。
- **R-S7 を cause-agnostic 化。** 陳腐化トリガーを「割当変更」から「生きた予測の完了が凍結スロットと乖離(原因＝割当/c のいずれでも)」へ一般化。これにより v11→v12 の「α_i にトリガー条件は存在しない」開示を**撤回**(記録された c 変更が凍結ベースラインを無効化する場合を検知。記録と実態の drift は依然非検知＝P0+R-U14)。
- **c は slot 次元/PV(t) 時間配分に効き、予算総額には効かない。** c=0 は定義域内(祝日/休暇)。恒久離脱は割当解除(§2.4)、一時的不在は c=0——境界は P0(モデルは規定しない)。
- **二層データを正直に開示。** A2「唯一のデータ＝4イベントログ」はノードのワークデータを指し、構成入力 c は別層(R-U14 履歴)。v12 で α_i について既に成立していた区別の一般化。
- **残余開示(§7)**: drift 非検知 / 暦未指定日の楽観既定 / 広範 c 変更のスロット陳腐化洪水 / 過去予測再現の valid-time / 恒久 c=0 の二経路。いずれも P0+実装に委ねる。

独立採点者(moira-gate-judge)の残存 Critical/Important = 0。新公理・新イベント・新状態・新要件番号なし(R-S7/R-T3 は意味論拡張)。

## 確定済みの分岐(v13 → v14: 警告の持続・消滅意味論)

> Moira 実装の画面アーキ(行為面=decision インボックス採用)検討中に独立敵対ゲートが MODEL 本体の穴を検出。
> moira-model-update の敵対ループ(adversary ×3 + fact-checker → ユーザー裁定 FORK → gate-judge)を経て v14 に確定。

- **警告は「現在の導出状態に対する述語」**であり、条件が真である限り可視ギャップに残り、**条件を偽化する入力(4イベント追記 または c 変更; R-S2)**が起きた時のみ消える(§2.1 新設条項)。歴史的事実そのものでは判定しない(R-U12=現行 latest-wins 値、P5 at-risk=implemented 再到達前)。
- **「acknowledge(イベント無し)では消えず、commit(イベント)で消える」。** 人間が見て受容し何もしないのは正当な**非コミット**で、警告を消さない(true な信号の隠蔽は §0/P0 に反する)。R-U13 事後合意・R-C3 辺操作・R-T4 スコープ削減は commit=イベントゆえ消える(整合)。
- **【ユーザー裁定 FORK】confirm/受容型と S/N の扱い → α(acknowledge 可変状態を持たない)を選択。** 具体ケース(夏季休暇で R-S7 が100スロット陳腐化/難研究の R-S3 thrashing/R-T4 再交渉中)付きで α/β/γ を AskUserQuestion に提示。ユーザーは **α＋「β 上位互換パスを §7 に開示」**を選択。
  - α: 警告は真の限り可視ギャップ会計に残す。提示層は「据え置き・既知/新規」を区別(畳む・淡色化)してよいが**会計から除いてはならない**(falsifiable な線=P0)。
  - β(将来の上位互換): 受容の監査証跡(誰がいつ何を据え置き受容したか)が UC 化したら、c(R-U14)と同型の**追記専用 acknowledgment 層**を後付けし、活性述語を「条件が真 ∧ 最終条件変化以降に未確認」へ拡張(ノードのワークデータでない別層ゆえ A2 非違反)。
- **R-S7 の「確認」=据え置き受容(非コミット)で陳腐化標識を解除しない。** 乖離は variance として正直に残す(EVM 本義;fact-checker が PMI/ANSI-EIA748 で CONFIRMED)。消滅は理由付き再ベースライン(イベント)か生きた予測の再収束のみ。一括確認=提示 salience、一括再ベースライン=イベント群。
- **「持たない」根拠の訂正(自己の過去前提の撤回):** acknowledge を持たない根拠は **A2 ではなく最小性**。acknowledge はノードのワークデータでないため A2 は直接禁じない(c と同じく別層たりうる)——v14 ループ中に敵対者が指摘し訂正。

独立採点者(moira-gate-judge)の残存 Critical/Important = 0。新公理・新イベント・新状態・新要件番号・新原理番号なし(既存条項の意味論明確化)。fact-checker NO_OBJECTION。

## 確定済みの分岐(v15 → v16: 実行カバレッジの正典化・獲得ルールの却下)

> ユーザー指摘「二値 EV(完了まで0→満額)だと執行中の進捗把握が遅れる」を起点。
> moira-model-update の敵対ループ(round1 P-EARN 却下 → ユーザー裁定 FORK → round2 P-EXECCOV → R-U8 反証の
> 再反論 → gate-judge PASS)を経て v16 に確定。思考実験 = `moira/thought-experiments/11-progress-granularity-earning-rule.md`。

- **当初案 P-EARN(`implementing` で固定割合 F の partial EV を稼ぐ獲得ルール)を却下。** Round1 で3敵対者が独立に
  「F は状態→重み写像で R-U8『no state-weight table』/P1『重みテーブルなし』該当」「EV に折り込む乗率で L47/§7#11
  (留保率と算術同型)抵触」「分解(P1)で除去可能」「PV 非対称・下方再見積で EV%>1 等の未処理多数」を指摘。
  fact-checker は 0/100・50/50・20/80 が標準 EVM の客観手法と NO_OBJECTION で裏取りしたが、Moira の L47/R-U8 は
  標準が保持する重み表を意図的に溶かした分**より厳格**ゆえ支えにならないと確認。著者は強行せず却下(accuracy-first)。
- **【ユーザー裁定 FORK (canon-home)】「執行中の集約可視化」の実現 → 正典 R-S 要件を選択**(他候補:実装/UI のみ・
  §7 開示)。partial EV を使わず核心会計を読み変えない別建てオーバーレイ **実行カバレッジ executionCoverage** を
  新要件 **R-S8** として追加。
- **executionCoverage = count-based coverage(scheduleCoverage 同型)。** = |合意済み有効葉のうち implementing| /
  |合意済み有効葉|(ノード数比率・分母0→0)。述語が「scheduled?」→「implementing?」に変わっただけで scheduleCoverage
  (R-S6)と構造同型。**予算に状態別乗率を掛けず状態述語でノードを数える**ゆえ R-U8 の射程外(R-U8 は EV% 稼得の
  state-weight を禁ずる)、**EV_abs/EV%/PV/SPI/CPI の式に一切触れない**。
- **R-U8 該当(FORK1)を scheduleCoverage 構造同型で反証 → 別 adversary の再反論で健全と確認(round3)。** でなければ
  R-S6/P2 自体が R-U8 違反になり正典が自己矛盾、という対称性が決め手。
- **de-rate 規律(必須)。** execCov は仕掛中の*量*であって出来高ではない(implementing に居座るだけのタスクも算入)。
