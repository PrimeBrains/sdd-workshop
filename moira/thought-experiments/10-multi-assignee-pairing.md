# 思考実験 10:一タスクへの複数アサイニー(ペア/モブ)と平準化の取りこぼし

> 問い:二人の開発者が**一つのタスクにペアで**就く(ペアプロ/モブプロ)。Moira はこれを表現できるか。割当・スケジュール・1.0MD/日平準化・容量会計はどう動くか。
>
> この思考実験は MODEL §2.4・R-T5 と P7/R-U11/R-T3 の接合部に**穴を露呈させた**(MODEL 自体の修正は後の moira-model-update に委ねる。本書は露呈に徹する)。以下は MODEL v10 を前提とする。

## 素朴な期待と、それが破れた点

最初は「割当は人間が与える入力(§2.4)なのだから、二人を割り当てればよい」と期待した。だが MODEL の割当機構は**単一の被割当者**を前提する。

- §2.4 L85:「割当は**作業に入るライフサイクル遷移の被割当者として記録**される(…が assignee を名指す)」——**被割当者(assignee)は単数**。
- §2.4 L85:暫定割当の変更は「被割当者を改めて名指す `transition` を追記して行う(append-only。**`(ts,id)` 順で最新の名指しが現行の割当**)」——新たな名指しは**置換**であって追加ではない。P1 を名指した後に P2 を名指すと、現行の割当は P2 **だけ**になる。
- R-T5 L269 も「被割当者を作業開始ライフサイクル遷移の属性として記録」と単数で書く。

つまり**「最新の名指しが現行の割当」という (ts,id) latest-wins 機構は、一タスクに同時に二人の被割当者を持たせる構造を持たない**。ペア/モブ——一つの作業単位に複数人が同時に就く——を割当・スケジュール軸で表現できない。これが素朴な期待の破れた点。

## 分析本体:cost 軸は二人を表せるのに、割当・計画軸は表せない

### 各導出での振る舞い

| 導出 | ペア(P1+P2 が一タスク T に就く)での振る舞い | 根拠/沈黙 |
|---|---|---|
| **割当 / ready** | T の被割当者は単数。P1 を名指し、次に P2 を名指すと、(ts,id) 最新の P2 が**現行の割当**になり P1 は外れる(§2.4 L85)。**二人を同時に現行割当として表す構造が無い** | §2.4 L85、R-T5 L269(単数 assignee) |
| **スケジュール / 平準化** | T は単一 assignee の 1.0MD/日平準化(P7/R-U11)に載る。**現行割当の一人(例 P2)の容量にしか計上されない。** もう一人(P1)が T に費やす実時間は、T の assignee でないため**平準化から漏れる** | P7 L158-160、R-U11 L212(人間ごとに平準化) |
| **コスト / AC** | cost は actor へ帰属し加算(P3 L147)。P1 と P2 が各々 `cost` イベントを T に計上でき、**AC は二人ぶんを正しく記録する**(03 TE L36 の「容量の分割」と同型) | P3 L147 |
| **過負荷(R-T3)** | R-T3(L263)は「割当が 1.0MD 平準化に収まらない人物・期間」を過負荷として警告。だが**P1 は T の assignee でない**(現行割当は P2)ため、P1 のペア作業は割当として平準化に入らず、**P1 の過負荷を R-T3 が捉えない**(cost は立つが割当は無い) | R-T3 L263(割当ベース) |

### 会計軸と計画軸の非対称

ここに鋭い非対称がある:

- **cost 軸(AC、P3)は複数人を表せる**——同一タスク T に P1・P2 それぞれの `cost` イベントが帰属でき、容量の実消費は二人ぶん記録される。
- **割当・計画軸(§2.4/R-T5/P7)は単一 assignee しか持たない**——T のスケジュール上の容量は現行割当の一人ぶんにしか計上されず、もう一人の容量消費は平準化(P7)・過負荷検知(R-T3)から漏れる。

結果、**ペアで 2.0MD/日を投じている作業が、スケジュール上は 1.0MD/日(一人)としてしか現れない**。生きた予測スケジュール(P7/P8)は実態より楽観に振れ、第二者の容量は「使われているのに割当として見えない」状態になる。

なお A4(L31)は「システムは人間の能力(スキル・習熟度)をモデル化しない」とするが、これは**能力(capability)**の非モデル化であって、「一作業単位に何人が就くか」という**割当の多重度**の話ではない。A4 はこの穴を塞がない。回避策としてタスクを二分割する(各人に一つ)ことは考えられるが、それは**一つの協働作業単位を二つに割る**ことで、同一単位への同時協働(ペアの本質)を表現するものではない。

## 見つかった穴

- **[Important] §2.4(L85)/ R-T5(L269)** — 被割当者は**単数**であり、(ts,id) latest-wins は「最新の名指しが**現行の割当**」=置換であって追加。**一タスクに複数の同時被割当者を表現する構造が無い**。ペア/モブ(一作業単位への同時複数人)は割当・スケジュール軸で表現不能で、**「単一 assignee」は明文化されていない暗黙の前提**になっている。/ V3(単一 assignee の隠れ前提)・V6(複数アサイニーの未処理ケース)。
- **[Important] P7(L158-160)/ R-U11(L212)/ R-T3(L263)× P3(L147)** — **会計軸(cost/AC)は複数人を表せる**(P1・P2 個別の cost イベント)のに、**計画軸(割当・平準化)は単一 assignee しか持たない**ため、ペアの第二者の容量消費が平準化(P7)から漏れ、生きた予測が楽観に振れ、R-T3 の過負荷検知が第二者を捉えない。同一作業に対する cost 軸と assignment 軸の表現力の非対称。/ V6(未処理ケース)・V2(P3 の actor 多重 vs §2.4 の単一 assignee)。

これら二点を辿ると、割当という人間入力(§2.4)が、現実に広く行われるペア/モブ作業——一作業単位への同時複数人——を計画軸で表現できず、cost 軸との非対称を残すことが分かり、**穴が見つかった**(網羅の証明ではない;MODEL §5・P2 と同じ留保)。

*Assignment records a single assignee (§2.4 L85 / R-T5 L269) and (ts,id) latest-wins makes the latest naming THE current assignment — a replacement, not an addition — so a task cannot carry two concurrent assignees. Pairing/mobbing (multiple people on one work unit) is therefore unrepresentable on the assignment/schedule axis; "single assignee" is an unstated assumption. Worse, the cost axis (P3) CAN record both actors while the planning axis (§2.4/P7/R-T3) cannot, so a pair spending 2.0MD/day appears as 1.0MD/day on the schedule and the second person's load escapes leveling and the over-allocation alert (R-T3).*

## レビュー来歴

独立敵対レビュー(自己検証ループ:fact-checker + 反転 adversary×3 + gate-judge)でゲート通過(誤引用0・論理破綻0・両 finding が論破を生存)。3 体の反転 adversary とも、一ノードへの同時複数被割当者を許す本文も、cost(actor 多重可)を平準化/過負荷検知へ橋渡しする本文も MODEL から引けず refuted=false(§2.4 L85 の latest-wins は単数収束する置換機構)。なお敵対者の一指摘「ペアを一ノードで表す UC を MODEL が要求するか(最小性原理 §5 との関係)」は、TE 本体の破綻ではなく **MODEL の設計分岐**(ペアの扱いを構造で持つか、§2.1 L56-58 のコミット境界として構造外へ委ねるか)であり、moira-model-update での裁定に委ねる FORK 候補として記録する(本書は露呈に徹する)。
