# 思考実験 09:並行する合意の衝突と人間権威の裁定

> 問い:見積の `proposed → agreed` は人間のみが行う(I6/R-U4)。だが現実のチームには人間が複数いる。**二人の人間が同一の見積ノードに、異なる凍結値で並行して `agreed` を発行**したら、Moira はどちらを採るのか。唯一のマージ規則 `(ts,id)` latest-wins(I3)で「最後に並んだ方」が勝つことは、計画入力ならともかく、コミットメントを伴う**権威判断**(§2.1)の競合解決として妥当か。
>
> この思考実験は MODEL I6/R-U4・A5・§2.1 と I3/§2.4 の接合部に**穴を露呈させた**(MODEL 自体の修正は後の moira-model-update に委ねる。本書は露呈に徹する)。以下は MODEL v10 を前提とする。

## 素朴な期待と、それが破れた点

最初は「合意は人間のみ(I6)と決まっているのだから、権威は明確」と期待した。だが MODEL を読むと、**「人間」は単一の権威クラスとして扱われ、人間どうしの競合を解く規定が無い**。

- I6(L124)「`proposed→agreed` の行為者は `human`」、R-U4(L192)「見積を `agreed` へ遷移させる行為を人間にのみ許可」(R-U4 ラベル/英文 EARS は L191、和文は L192)。**いずれも「人間か否か」の境界(対エージェント)を引くだけで、「どの人間か」「複数人間の競合」は語らない。**
- A5(L34)は「**権限において非対称**(見積の確定=合意は人間のみ)」とする。これは人間とエージェントの非対称であり、**人間集合の内部の権限分配ではない**。
- §2.1(L49, L54)は合意を「AI が提案、人間が確定」「希少資源たる人間判断を**意図的に投下する点**」と、コミットメントを伴う重い判断と位置づける。
- 一方、競合を解く規則は I3(L121)「event id は大域ソート可能、`(ts,id)` で決定的マージ」しか無い。§2.4(L85)は割当について「`(ts,id)` 順で最新の名指しが現行の割当」と、この latest-wins を**取り消し可能な計画入力**に適用している。

つまり**合意の競合は、構造上 I3 の `(ts,id)` latest-wins に委ねられる**ことになる。これが素朴な期待の破れた点:権威判断の競合が、取り消し可能なデータと同じ機械的順序で解かれてよいのかが問われていない。

## 分析本体

### 二人が異なる値で同時に合意する

開発者 P1 と P2 がともに人間で、同一見積ノード N に対し:
- P1 が N を `agreed`、凍結値 8MD(R-U7 L201「理由付き凍結値」;R-U7 ラベル/英文 EARS は L200)。
- P2 が(ほぼ同時に)N を `agreed`、凍結値 13MD。

両者とも I6/R-U4 を満たす正当な `proposed→agreed` 遷移である(ともに human)。

| 観点 | 起きること | 根拠/沈黙 |
|---|---|---|
| **どちらが現行か** | I3 で `(ts,id)` が後の方が「現行の合意値」になる。同一 ts なら id の大小という**実質的に任意の順序**で決まる | I3 L121 はマージの決定性を保証するが、**権威の正しさ**は保証しない |
| **これは権威判断として妥当か** | §2.1 L54 は合意を「希少資源たる人間判断を意図的に投下する点」とする。だが latest-wins は「最後に書いた人が勝つ」であり、**P1 と P2 のどちらに権威があるか**(リードか否か、オーナーか否か)をモデルは一切問わない | A4(L31)が人間の能力・属性をモデル化しないため、権威属性も持たない |
| **競合の可視化** | P5(L151)は EV 非単調を「後退遷移/上方再見積/新規合意」の三因で可視化するが、これは**値の変化**を映すもの。**「権威ある二者が矛盾合意した」という競合自体を検知・可視化する導出は無い** | P5 は値の動きを映すが権威競合を映さない |

### 「人間=単一権威」は隠れ前提か、意図的なスコープ外か

ここに二つの読みが並立し、MODEL はどちらとも明言しない:

1. **意図的スコープ外説**:A4(L31)は「システムは人間の能力(スキル・習熟度)をモデル化しない…硬いルールで縛らない」とする。権威・オーナーシップも「人間の組織的領域」であり、モデルは持たない——という読み。この読みなら、latest-wins は単に「最新の人間合意が現行」という append-only の自然な帰結で、誰が権威かは組織(P0 のコミット判断)に委ねる、となる。
2. **未処理ケース説**:I6/R-U4 が「人間のみ」と**権威境界を明示的に引いている**以上、その境界の内側(人間どうし)の競合解決も本来モデルの関心であり、それを I3 という**普遍的マージ規則**(L121;latest-wins を「現行値の選択」として明示適用する先例は §2.4 の割当=取り消し可能のみ)に暗黙に委ねるのは、権威判断(§2.1)への適用妥当性を検証していない、という読み。

**MODEL はこのどちらの立場も明記していない。** 「合意の競合は latest-wins で解く(そう決めた)」とも、「人間内部の権威はスコープ外(A4 の延長)」とも書かれていない。§2.4 が割当(取り消し可能)に latest-wins を**明示適用**しているのに対し、合意(コミットメント)への適用は**沈黙**のまま構造的帰結として残る。

この沈黙が穴である。なお、二つの読みの**どちらを採るか**は「Moira が人間の権威をモデル化する意図を持つか」という**設計者の意図に属する分岐**であり、本実験は露呈に徹してその裁定はしない(moira-model-update での裁定対象、必要ならユーザーへの FORK)。

## 見つかった穴

- **[Important] I6(L124)/R-U4(L191-192)/A5(L34)× I3(L121)/§2.1(L54)** — モデルは合意の権威境界を「人間か否か」で引く(I6/R-U4/A5)が、**複数の人間が同一ノードに矛盾する `agreed` を並行発行したときの競合解決を、権威判断として規定しない**。I3 `(ts,id)` latest-wins は全イベント横断の普遍的マージ規則(L121。`proposed→agreed` も §2.8 L113 で transition=I3 の対象)で、並行 `agreed` も機械的に確定はするが、**それがコミットメントを伴う権威判断(§2.1 L54)に妥当かは未検証**——latest-wins を「現行値の選択」として明示適用する先例は §2.4 の割当(=取り消し可能入力)のみで、合意への適用は沈黙のまま構造的帰結として残る。同一 ts なら id の任意(=権威的に無意味な)順序で勝者が決まる。/ V6(未処理ケース)・V2(I3 の機械的マージ vs §2.1/A5 の権威性)。
- **[Important] A4(L31)× I6/R-U4** — 「人間」を単一の権威クラスとして扱う前提が、明示の disclaimer 無しに置かれている。A4(人間の属性をモデル化しない)を延長すれば「人間内部の権威はスコープ外」とも読めるが、I6/R-U4 が権威境界を明示している以上その内側の沈黙は隠れ前提。**「人間内部の権威・競合をモデル化する意図があるか」をモデルが明言していない**(この択一は設計意図に属する分岐=FORK 候補)。/ V3(「人間=単一権威」の隠れ前提)。

これら二点を辿ると、合意という MODEL の中核的な権威判断が、**複数人間の競合に対して沈黙し、データマージ規則に暗黙委任されている**穴を残すことが分かり、**穴が見つかった**(網羅の証明ではない;MODEL §5・P2 と同じ留保)。

*The model draws the agreement-authority boundary at human-vs-agent (I6/R-U4/A5) but is silent on conflict among multiple humans: two humans concurrently agreeing different frozen values are both valid, and the universal merge rule I3 `(ts,id)` latest-wins (L121; its only explicit current-value application is §2.4 assignment, which is revisable) mechanically resolves them — yet whether that is valid for a commitment-bearing authority decision (§2.1) is unexamined, with same-ts ties broken by arbitrary id order. Whether intra-human authority is in scope (an A4 extension) or an unhandled case is left unstated — a design-intent fork.*
