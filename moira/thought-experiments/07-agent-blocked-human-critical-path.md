# 思考実験 07:長時間エージェント作業に律速される人間タスクとクリティカルパス

> 問い:ある人間タスク H が、長時間かかるエージェントタスク A に DAG 依存している(A → H)。H は A が閾値(R-D2 既定 `implemented`)を満たすまで `ready` になれない=実際には A の完了を待ってブロックされる。だが P7 は「クリティカルパスは人間タスクの連鎖」「エージェント作業は…人間接点の間隙を埋める」と言う。エージェント作業が人間作業を**律速**するこの逆転局面で、導出スケジュールと期日アラート(R-T4)は正しいか。
>
> この思考実験は MODEL P7(L158-160)・R-T2・R-T1 と R-T4 の接合部に**穴を露呈させた**(MODEL 自体の修正は後の moira-model-update に委ねる。本書は露呈に徹する)。以下は MODEL v10 を前提とする。

## 素朴な期待と、それが破れた点

最初は「エージェントは速く・遊休可なので、人間作業の合間を埋めるだけ。スケジュールの律速は人間で正しい」と期待した。P7(L159)はまさにこう書く——「エージェント作業は平準化対象外でスラックを持たず、**人間接点の間隙を埋める**。クリティカルパスは人間タスクの連鎖。」

だがこの記述は**エージェント作業が人間作業より短く、間隙に収まる**ことを暗黙に前提する。現実にはエージェントタスクが長時間化する局面がある——思考実験03(L82)自身が「夜間にエージェントが自律で進めた作業」「眠らないため、運用上は夜通し作業を進められる」と認める。エージェント作業は無時間ではない:P6(L156)が「ts 差はリードタイム」と定めるとおり、エージェントタスクも実時間(リードタイム)を消費する。

つまり**A が長時間で H が A に依存するとき、A は「間隙を埋める」のではなく、A 自身が H の前に立ちはだかる律速区間になる**。これが素朴な期待の破れた点。

## 分析本体:エージェント区間がクリティカルパスから落ちる

### P7 内部の二つの記述が衝突する

P7(L158-160)は二つのことを言う:
1. 平準化(1.0MD/日)は**人間の割当**に対して行う(R-U11 L212 とも一致:人間のみ平準化、エージェントは制約対象外)。
2. **クリティカルパスは人間タスクの連鎖**であり、エージェント作業は間隙を埋める。

記述(1)は「資源平準化の対象は人間だけ」という資源面の主張で、これは妥当。だが記述(2)は**クリティカルパス(プロジェクト所要期間を決める最長依存連鎖)が人間タスクで構成される**というより強い主張で、A → H のように**エージェント区間が依存連鎖の一部かつ長時間**の場合に成り立たない。

| 導出 | A → H(A 長時間)での振る舞い | 根拠/沈黙 |
|---|---|---|
| **ready 判定** | H は A が `implemented`(R-D2 既定)になるまで `ready` にならない。**H の開始は A の完了リードタイムに律速される** | R-D1/R-D2 は正しく H をブロックする |
| **クリティカルパス長** | P7(L159)が「クリティカルパス=人間タスクの連鎖」とするなら、A のリードタイムはパス長に算入されない恐れ。R-T2(L261)は「エージェント作業を…人間接点間の実行可能スパンとして**表示**」とするが、**それがクリティカルパス計算(R-T1 のクリティカルパス優先)に算入されるとは書かれていない** | P7 L159 と R-T1/R-T2 が、A の duration をパスに含めるか silent |
| **期日アラート(R-T4)** | R-T4(L267)は「導出スケジュールが外部期日を超える場合アラート」。だがクリティカルパスが A のリードタイムを過小評価していれば、**導出スケジュールが楽観に振れ、本来超過する期日を超過と判定しない**(偽陰性) | R-T4 は導出スケジュールの正確性に依存するが、その正確性が P7 の前提で崩れる |

### 「間隙を埋める」モデルが破れる具体

P7 L159「エージェント作業は…人間接点の間隙を埋める」は、人間タスク群がスケジュールの骨格で、エージェントがその隙間に収まる像を描く。だが A → H では:

- A は H の**前提**であり、H は A 無しに始まらない。A は「隙間を埋める」のではなく、**H の開始時刻を後ろにずらす律速要因**。
- もし A のリードタイムが、その前の人間タスクと H の間の「間隙」より長ければ、A は間隙に収まらず、**スケジュール全体を押し出す**。
- このとき真のクリティカルパスは「…→ A → H → …」で **A を含む**が、P7 L159 の「人間タスクの連鎖」という規定はこれを捉えない。

MODEL は R-U11(エージェント非平準化)と P7(人間律速)を資源面の正しい主張として持つが、**「エージェント区間が依存連鎖上で人間タスクを律速する」時間面の扱いを定義していない**。

## 見つかった穴

- **[Important] P7(L159)× R-T1(L257)/R-T2(L261)** — P7 の「クリティカルパスは人間タスクの連鎖」「エージェント作業は人間接点の間隙を埋める」は、エージェント作業が短く間隙に収まることを暗黙の前提とする。だが A → H(エージェント A が長時間、人間 H が依存)では A が依存連鎖上の律速区間になり、**A のリードタイム(P6 L156 で実時間を持つ)がクリティカルパスに算入されるか未定義**。R-T2 はエージェント作業の「表示」を定めるのみで、R-T1 のクリティカルパス計算への算入を規定しない。/ V2(P7 内部:資源面「人間のみ平準化」と時間面「人間が律速」の混同)。
- **[Important] R-T4(L267)× P7(L159)** — 上の過小評価が成り立つと、導出スケジュールが楽観に振れ、R-T4 の期日超過アラートが**偽陰性**(本来超過するのに警告しない)になりうる。長時間エージェント作業が常態化する運用(03 L82 が認める夜間自律作業)でスケールするほど顕著。/ V4(スケール:エージェント長時間化)。

これら二点を辿ると、P7 の「人間律速」モデルが、**エージェント作業が依存連鎖上で人間を律速する逆転局面を扱えず**、期日導出の信頼性に穴を残すことが分かり、**穴が見つかった**(網羅の証明ではない;MODEL §5・P2 と同じ留保)。

*P7 says "critical path = human chain" and "agent work fills gaps," presupposing agent work is short enough to fit between human touchpoints. But when a human task H depends on a long agent task A (A→H), A is a rate-limiting segment on the dependency chain, with real lead time (P6). Whether A's lead time enters the critical-path computation (R-T1) is undefined — R-T2 only "displays" agent spans — so the derived schedule can be optimistic and R-T4's deadline alert can be a false negative.*
