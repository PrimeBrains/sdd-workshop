# 思考実験 05:割当済みだが未合意のタスク —— 半凍結ベースラインと PV/SPI/I4 の破綻

> 問い:あるタスクに人間が割当を与え(R-T5)、`→implementing` へ進める。だがその見積はまだ `proposed` のまま(人間が `agreed` していない)。**着手(時間軸)と合意(価値軸)は直交する**(P0)ので、この「割当済みだが未合意」は構造上ありうる。スケジュールに載って**スロットは凍結**されるが、**予算は未合意ゆえ凍結されない**——この「半凍結ベースライン」のとき PV・SPI・I4 はどう振る舞うか。
>
> この思考実験は MODEL §3(導出指標 L165–173)・I4・R-U8 の射程の接合部に**穴を露呈させた**(MODEL 自体の修正は後の moira-model-update に委ねる。本書は露呈に徹する)。以下は MODEL v10 を前提とする。

## 素朴な期待と、それが破れた点

最初は「EV_abs は R-U8 が未合意を除外し、容量は生きた予測スケジュールが拾うので、割当済み・未合意も二会計面で閉じる」と期待した。実際、二つの軸は閉じる:

- **価値軸**:R-U8(L203-204)が「EV_abs = Σ(completed frozen estimates) — **excluding un-agreed and un-estimated work** / 未合意・未見積を除外」と要件として定めるので、未合意の完了は EV_abs に乗らず(=除外)、EV% ≤ 1 は順序非依存に保たれる。低カバレッジは R-S4 が de-rate する。
- **時間軸(生きた予測)**:最新値(latest)が予測を駆動する(§3 L172)ので、proposed のタスクも生きた予測スケジュールに載り容量を消費し、収まらなければ R-T3 が過負荷を出す。

だが**第三の会計面=凍結ベースライン(PV/SPI/PMB)**を見落としていた。MODEL は「**凍結ベースライン(PV/SPI 用)と生きた予測スケジュール(P7/P8…)は別物**である」(§3 L173)と明記しており、**R-U8 の射程は EV_abs/EV% のみで、PV・I4・SPI を支配しない**。ここで穴が開く。これが素朴な期待の破れた点。

## 分析本体:スロットだけ凍結し予算が凍結しない

### この局面は構造的に到達可能

- スケジュールに載る唯一のゲートは**割当**であって合意ではない(§2.4 L87:「割当のないタスクは…スケジュールに載らない」=割当があれば載る)。`ready` の契機も辺ポリシーの充足で合意非依存(R-D1 L282、思考実験01)。
- ゆえにノードは `proposed`(未合意)のまま `ready → implementing` に進み、スケジュールに載れる。状態機械(§2.5 L90)に合意ゲートは無い。

### 二次元のベースラインが非対称に凍結する

§3 のベースライン(PMB)は二次元が別々の契機で確定する:

- **① 値(予算)**:「見積ノードが `agreed` になった時、その凍結見積値で確定」(§3 L166①)。**未合意なら予算は凍結されない。**
- **② 計画スロット**:「当該サブ単位が**初めてスケジュールに載った時**(初回の割当が付き P7 平準化に入った時)の完了予定で確定」(§3 L166②)。「より確度の高い着手(`→implementing`)時点で凍結する」契機も明示される(§3 L168)。**この凍結は合意を前提としない。**

つまり割当済み・未合意ノードは**スロットは凍結・予算は未凍結という「半凍結ベースライン」**を取る。

### 各導出での破綻

| 導出 | 半凍結ベースラインでの振る舞い | 根拠/沈黙 |
|---|---|---|
| **EV_abs** | R-U8 が未合意を除外 → 寄与 0(価値軸はここは閉じる) | R-U8 L203-204 |
| **PV(t)** | 「PV(t) = ベースライン上その時点で**完了予定**のサブ単位の**ベースライン予算**総和」(§3 L169)。**この「スケジュール済み・未合意」象限の PV 包含を MODEL が規定していない**。二読あり、いずれも欠陥:読みA「スロットを持つので『完了予定』に入る」→予算が無く**被加算項が未定義**;読みB「ベースライン上=予算×スロットの対なので入らない」→だがこのノードは実スロットを消費する(下行)のに PV に**0 しか寄与しない**。L169 の除外節は「**合意済みだが未スケジュール**」という**逆象限専用**で当象限を扱わず(むしろ「スケジュールに載った時点で PV に算入」)、どちらの読みも MODEL に接地しない | §3 L169(逆象限のみ規定) |
| **SPI** | 「SPI = EV_abs / PV」(§3 L171)。分子 EV_abs は R-U8 で**除外(0)**。分母 PV は上記のとおり未規定——読みA なら被加算項が未定義で SPI が ill-defined、読みB なら実スロットを消費したノードが PV に 0 寄与で**分母が実態より過小→偽のスケジュール信号**。いずれの読みでも SPI が壊れる | §3 L171 |
| **I4 完了施錠** | 「完了サブ単位のベースライン寄与(**予算・スロット**)は完了時点で施錠」(I4 L122)。**I4 は合意を前提しない**(対照的に I1 L119 は v10 で「合意済みの子」へ条件化されたが、I4 は未条件化)。R-S1(L237-238)も無条件。だが予算次元が未確定なので**施錠対象の予算が空(空虚な施錠)**になり、しかも下流の R-U8/EV_abs(L170)はこの完了寄与を無視する——**I4(無条件施錠)と R-U8(寄与除外)の間の沈黙の不整合** | I4 L122、R-S1 L237-238 |
| **R-S6 デレート** | スケジュール・カバレッジは「**合意済み作業**のうちスケジュール済みの割合」(R-S6 L252-253)。未合意-スケジュール済みノードは**分母(合意済み)の外**にあり、R-S6 の de-rate でも拾えない。実スロットを消費し他作業を平準化で押しのけたタスクが、SPI 信号からも対の読みからも構造的に**不可視** | R-S6 L252-253 |

要するに、R-U8 は EV_abs/EV%(価値軸)を閉じるが、**PV・SPI・I4(凍結ベースライン面)を閉じない**。半凍結ベースラインは「スロットあり・予算なし」で PV 被加算項を未定義にし、SPI を分子分母非対称にし、I4/R-S1 に存在しない予算の施錠を命じる。

### 偽の普遍命題がこの穴を覆っている

P1(L140)は「**完了サブ単位は必ず合意(`agreed`)を経て完了するため** EV_abs は合意済み領域に収まり、EV% ≤ 1 が保たれる」、L141 は「Completed sub-units are always `agreed` first」と**普遍量化**で断定する。だが状態機械(§2.5 L90)に合意ゲートは無く、`ready` は辺ポリシー充足で合意非依存(R-D1 L282、本実験の前提)。**本実験のシナリオ(proposed のまま完了)が、この「必ず/always」を直接反証する。** EV% ≤ 1 自体は R-U8 が独立に救うが、P1 のこの偽の普遍命題が「未合意完了時に『凍結見積=ベースライン予算』が何を指すか」という PV/I4 の穴を覆い隠している。

## 見つかった穴

- **[Critical] §3 L166②/L168(スロットは合意非依存に凍結)× L166①(予算は合意時のみ凍結)× L169(PV 除外は逆象限のみ)× I4 L122 × R-U8 L203-204** — 割当済み・未合意ノードは**半凍結ベースライン**(スロット凍結・予算未凍結)を取るが、**この「スケジュール済み・未合意」象限の PV/SPI/I4 の扱いを MODEL が規定していない**(L169 の除外節は「合意済みだが未スケジュール」の逆象限専用)。PV 包含は二読あり**いずれも欠陥**:読みA(算入)では被加算項が未定義、読みB(非算入)では実スロットを消費したノードが PV に 0 寄与で SPI 分母が過小→偽信号。SPI=EV_abs/PV(L171)は分子(R-U8 除外)と分母(未規定)が噛み合わず、I4(L122)/R-S1(L237-238)は無条件施錠を語るが予算次元が空で、下流 R-U8/EV_abs(L170)はその寄与を無視する(I4 と R-U8 の沈黙の不整合)。R-U8 の射程は EV_abs/EV% のみで PV/SPI/I4 を支配しない。/ V2(L166② 合意非依存スロット vs L166① 合意依存予算 vs L169 片象限除外)・V6(scheduled-but-unagreed 象限の未処理)。
- **[Important] P1 L140/L141** — 「完了サブ単位は必ず合意を経て完了する/always `agreed` first」は**偽の普遍命題**。状態機械(§2.5 L90)に合意ゲートは無く `ready` は辺ポリシー充足(R-D1 L282)なので、proposed のまま完了に到達でき、本シナリオ自体が反証する。EV% ≤ 1 自体は R-U8 が独立に救う(P1 のこの普遍命題に依存しない)ため P1 を撤回しても EV% 上界は保たれるが、**正典が自身の妥当性検証で覆る普遍命題を載せている**点が問題(上記 PV/I4 の未規定は P1 とは独立に残る)。/ V3(過剰主張/隠れ前提)。

これら二点を辿ると、着手⊥合意の逆方向(着手先行・合意未了)が、**凍結ベースライン面で PV/SPI/I4 を破綻させる**ことが分かり、**穴が見つかった**(網羅の証明ではない;MODEL §5・P2 と同じ留保)。

*Assigned-but-unagreed is structurally reachable (scheduling gates on assignment, not agreement — §2.4 L87, R-D1 L282), so the slot freezes agreement-independently (§3 L166②/L168) while the budget freezes only at `agreed` (L166①): a half-frozen baseline. R-U8 (L203-204) closes only the EV_abs axis, not the frozen baseline. MODEL leaves this scheduled-but-unagreed quadrant unspecified for PV/SPI/I4 — L169's exclusion clause covers only the opposite quadrant (agreed-but-unscheduled) — and either reading is defective: if the node is admitted to PV its baseline-budget term is undefined; if excluded, a slot-consuming node contributes 0 to PV and understates the SPI denominator (a false schedule signal). I4 (L122) / R-S1 (L237-238) freeze unconditionally over an empty budget dimension while downstream R-U8/EV_abs (L170) ignores the contribution — a silent I4-vs-R-U8 mismatch. P1's "completed sub-units are always agreed first" (L140/L141) is a false universal the scenario itself refutes (EV% ≤ 1 is independently saved by R-U8; the PV/I4 gap stands regardless).*

## レビュー来歴

自己検証ループ(fact-checker + 反転 adversary×3 + gate-judge)を3ラウンド経て、穴の所在を**収束**させた。

- **第1版**(EV%>1 か未定義)→ **論破**:R-U8(L203-204)が未合意を EV_abs から規範的に除外するため EV%>1 は構造的に不能、寄与は「除外=0」で定義済み。当初 R-U8 を見落とした誤読を是正。
- **第2版**(「モデルが処理する/破綻なし」へ反転)→ **論破**:堅牢性確認モードの adversary 3 体+gate-judge が、価値軸(R-U8)と時間軸(生きた予測)の二軸しか点検しておらず**第三の会計面=凍結ベースライン(PV/SPI/I4)**を見落としていると指摘。R-U8 は PV/I4/SPI を支配せず、scheduled-but-unagreed ノードは半凍結ベースラインで PV 項を未定義にする。「PV から定義上除外」は L169 の逆象限専用節を当象限に誤接続した捏造だった。
- **本版**(半凍結ベースラインの Critical/Important)→ 穴狙いモードの自己検証で**ゲート通過**(誤引用0・論理破綻0・両 finding が 3 体一致で refuted=false, high)。EV_abs 軸が R-U8 で閉じることは保持しつつ、真の穴(PV/SPI/I4)を正しく定位した。さらに一敵対者の精度指摘を反映し、PV 包含を「確定的に算入→未定義」から「**当象限が未規定で、二読いずれも欠陥**」へ、I4 を「存在しない予算の施錠を命じる」から「**空虚な施錠 + I4/R-U8 の沈黙の不整合**」へ精緻化した(穴の核は不変)。
