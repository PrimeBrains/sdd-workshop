# 思考実験 02:エンハンス要件への対応(supersede)

> 問い:あるエンハンス要件が生まれ、仕様駆動で対応する。有向グラフ(DAG)はどうなるか。見積ノードはどのノードとどのノードの間に挟まるか。
>
> この思考実験が **§2.7「supersede:エンハンスによる既存変更」** と **R-D7 / R-S5** を発見・確定させた。以下は MODEL v10 を前提とした到達点。

## 木構造:エンハンスは root 直下の新 feature

素朴案(エンハンスを既存 feature-X の子として足す)は崩れた。feature-X は既に tasks 承認済みで、**完了・受入済みのノードは施錠される**(I4;未完了ノードは理由付き再ベースラインで改訂可だが、完了済みの確定履歴は汚さない)。

だからエンハンス E は **root 直下の新しい feature `feat-E`** として立て、独自の `req-E → design-E → tasks-E → 実装` サイクルを回す。feat-E と既存 feature-X の関係は「木の親子(価値集約)」ではなく「DAG の辺」で表す。DAG 辺には二種別あり、**依存辺は時間制約**(ready 判定に使う)、**supersede 辺は価値・履歴関係**(ready 判定に使わない)である(A3/§2.7)。

## DAG の形状:feat-E から既存へ二種類の辺

- **第一に論理的前提の辺**:エンハンスは既存機能の上に成り立つので「既存が先」。既存 impl が既に accepted なら即 ready で、実質「前提の明示」。
- **第二に supersede(置換)の辺**(本思考実験の発見):エンハンスが既存 impl-X1 の振る舞いを**変更**する場合、impl-X1 を後退遷移で再オープンするのは誤り。後退遷移は「却下=失敗」の意味を持ち、エンハンス(失敗でない)が P5 のスラッシュ信号を汚す。正しくは新ノード `impl-X1'` を feat-E の子として立て、`impl-X1' →(supersede)→ impl-X1` の置換辺(向きは新→旧)を張る。既存ノードは accepted のまま不変。**置換辺は ready 判定に使わない**(価値・履歴関係;§2.7)——依存辺との機能差。

supersede は新イベントではなく `relate` が張る**辺の種別**(依存／置換)の一つ(R-D7)。4イベントの最小性を保つ。

## 見積ノードの位置(主問への一般則)

見積ノードは「ある作業ノードの**先行(predecessor)**として、見積の材料が揃った最初の地点に張られる」(est→作業 の向き;R-E1/§2.3 で est は作業フェーズに先行する)。

- `est(req-E)` は [feat-E の起点] と [req-E] の間。エンハンス特有なのは、ここが既存仕様のどこに触れるかの**影響調査**を兼ねること(重ければ独立 spike に分けてよい)。
- `est(design-E)` は [req-E(accepted)] と [design-E] の間。
- 実装の見積は `est(impl)` として **tasks-E の後**にある(tasks.md を入力に;§2.3)。生成順は「tasks 完了 → feature を主語とする decompose が実装ノードを**未見積のまま誕生**させ → est(impl) がそれを tasks.md を入力に見積もり → 人間合意でカバレッジ回復」(§2.3)。実装ノードは est(impl) の前に既に(未見積で)存在する点に注意。

すべての見積は「前段の成果物を入力に、人間と AI が擦り合わせ、人間が合意する」同型の営み。

## EV の扱い(累積保持 + 現行有効の分離)

supersede された旧 impl-X1 の **EV_abs(累積出来高)は過去の事実として basis に残る**(二重計上でなく、実際に二回働いた正直な反映)。これは cancelled(R-C2 で active basis から除外)とは異なる扱い。

ただし二つの読みを分離する(R-S5):**累積稼得**(EV_abs、supersede 済み含む)と**現行進捗**(現行有効集合=supersede されていない葉だけの EV%)。**カバレッジ(P2)と対で読むのは現行進捗(EV%)**であり、累積稼得 EV_abs は基底が異なるため coverage と独立に読む(§2.7)。ダッシュボードはこれらを区別表示し、陳腐化した旧機能を現行有効と誤読させない。

## 確認できたこと

エンハンスは「既存を壊さず、新 feature として前進的に積み、置換関係で既存と結ぶ」。これにより却下(後退)とエンハンス(前進)が意味論的に分離された。新公理・新イベント・新状態を要さず、**§2.7 / R-D7 / R-S5** の確定に至った。
